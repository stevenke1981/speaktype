# SpeakType - 語音輸入法（模組化開發版）

## 1. 專案概述
SpeakType 是一個高度模組化的 Windows 語音輸入法，支援全域熱鍵、PTT 錄音、情境模板切換，並使用 whisper-rs 進行高品質語音辨識。

## 2. 模組化開發原則
- 以函式作為主要模組化單位
- 每個函式職責單一、命名清楚
- 模組之間透過明確介面通訊
- 使用 `pub(crate)` 控制可見性

## 3. 推薦模組結構

```rust
src/
├── main.rs
├── app.rs
├── modules/
│   ├── audio/          # 錄音與音訊處理
│   ├── whisper/        # whisper-rs 模型與推論
│   ├── input/          # enigo 鍵盤輸入與熱鍵
│   ├── gui/            # egui 介面（含系統匣）
│   ├── scenario/       # 情境模板與語言切換
│   └── utils/          # 模型下載、裝置偵測、設定檔
```

## 4. 主要功能需求

### 4.1 裝置狀態顯示（主視窗）
在主視窗中顯示以下裝置就緒狀態：

- **麥克風**：就緒 / 未偵測到裝置
- **GPU**：顯示型號，例如 `GPU (RTX 3060 Ti)：就緒`
- **模型**：`large-v3-turbo` 已載入 / 載入中 / 未找到

狀態應在 App 啟動時自動偵測，並即時顯示。

### 4.2 核心功能
- 兩種錄音模式：PTT（按住）與觸發鍵
- 全域熱鍵支援
- 情境模板切換（聊天、寫作、程式碼）
- 自動中英文語言偵測
- 辨識完成後自動貼上文字
- 模型不存在時自動下載
- App 啟動時預載模型
- 歷史辨識紀錄

## 5. 技術堆疊
- GUI: egui
- 語音辨識: whisper-rs + large-v3-turbo（CUDA）
- 錄音: cpal
- 鍵盤輸入: enigo
- CUDA 偵測: cudarc / cuda-runtime-sys

## 6. 開發順序建議（模組化）
1. `utils` 模組（裝置偵測 + 模型下載）
2. `whisper` 模組（模型載入與推論）
3. `audio` 模組（錄音功能）
4. `input` 模組（熱鍵 + enigo）
5. `gui` 模組（主視窗 + 狀態顯示）
6. 整合與測試

## 7. 非功能需求
- 完全離線可用
- 啟動預載模型
- 無 GPU 時給予明確提示與 CUDA 安裝引導
